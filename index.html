      화면 구조 / 데이터 구조 / 권한 로직

      화면 구조

      • 업무관리 > 프로젝트 목록 > 프로젝트 상세
      • 상세 페이지는 탭이 아니라 스크롤형 단일 화면
      • 각 섹션은 카드형 컴포넌트로 분리
      • 좌측 바로가기 메뉴로 해당 섹션 이동

      데이터 구조

      • 프로젝트 기본정보
      • 회의록 리스트 + 상세
      • 배정인원(대분류/소분류/인원목록)
      • 관련메일 리스트
      • 수주일정 + 지연사유서 + 승인상태
      • 완료예정일 + 변경사유 히스토리
      • 납품차수 + 파일 + 다운로드 권한 상태

      권한 로직

      • 수주 후 즉시 착수 불가 시 사유서 작성 필요
      • 상부 승인 완료 후 착수일 입력 가능
      • 완료예정일 변경 시 사유 입력 필수
      • 납품자료 다운로드는 실장 승인 필수

      개발 권장사항

      • 프로젝트 ID 기준으로 회의록/메일/납품 연계
      • 변경 로그는 별도 테이블 또는 이력 컬렉션으로 저장
      • 다운로드 권한은 승인 상태와 사용자 역할을 함께 검사
      • 상세 페이지는 추후 API 연동 고려한 구조로 분리
